## Project Overview

An **Intrusion Detection System (IDS)** monitors network traffic and flags suspicious activity before it causes damage.
This project builds a machine learning-powered IDS using the widely studied **NSL-KDD dataset** to train classifiers
that can distinguish between normal network traffic and four major categories of cyber attacks.

The project includes a full pipeline:

```
Raw Dataset → Preprocessing → Model Training → Prediction → Interactive Dashboard
```

All code is modular, commented, and designed for beginners who want to understand how
machine learning is applied to real-world cybersecurity problems.

---

## Cybersecurity Relevance

Modern networks face thousands of attacks every day. Traditional rule-based systems cannot keep up with
evolving attack patterns. Machine learning-based IDS offers:

- **Adaptive detection** — learns patterns instead of relying on fixed rules
- **Attack classification** — distinguishes DoS, Probe, R2L, and U2R attack types
- **Scalability** — can be retrained on new data as threats evolve
- **Speed** — real-time predictions on incoming traffic records
- **Visibility** — dashboard gives security analysts instant threat insights

This project demonstrates how ML can support the work of a **Security Operations Center (SOC)**.

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/your-username/intrusion-detection-system.git
cd intrusion-detection-system
```

### 2. Create and activate a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate        # macOS / Linux
.venv\Scripts\activate           # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Download the NSL-KDD dataset

Download from [Kaggle NSL-KDD](https://www.kaggle.com/datasets/hassan06/nslkdd) and place these files in `data/`:

```
data/KDDTrain+.txt
data/KDDTest+.txt
```

## How to Run

Run these steps in order from the `intrusion-detection-system/` folder.

### Step 1 — Preprocess the dataset

Cleans the raw data, encodes features, normalizes values, and saves processed CSVs.

```bash
python preprocessing.py
```

Output files:
- `data/processed_train.csv`
- `data/processed_test.csv`

### Step 2 — Train and compare models

Trains Random Forest, Decision Tree, and Logistic Regression. Automatically selects and saves the best model.

```bash
python train_model.py
```

Output files:
- `models/intrusion_model.pkl`
- `models/label_encoder.pkl`
- `models/model_scores.csv`
- `charts/model_accuracy_comparison.png`

### Step 3 — (Optional) Generate visualization charts

```bash
python visualize.py
```

Output files in `charts/`:
- `attack_distribution.png`
- `confusion_matrix_heatmap.png`
- `normal_vs_attack_traffic.png`

### Step 4 — Run predictions from the command line

```bash
python predict.py
```

Prints prediction result, attack type, and confidence for sample traffic records.

### Step 5 — Launch the Streamlit dashboard

```bash
streamlit run app.py
```

Then open [http://localhost:8501](http://localhost:8501) in your browser.

## Model Comparison

Three classifiers are trained and compared automatically during `train_model.py`.

| Model | Strengths |
|---|---|
| Random Forest | High accuracy, handles imbalanced data well, resistant to overfitting |
| Decision Tree | Fast, easy to interpret, good baseline |
| Logistic Regression | Simple, fast training, works well on linearly separable data |

The model with the **highest test accuracy** is saved automatically as `intrusion_model.pkl`.

## Attack Categories

The NSL-KDD dataset uses five broad attack categories:

| Category | Meaning | Examples |
|---|---|---|
| `normal` | Legitimate network traffic | Regular HTTP, FTP sessions |
| `dos` | Denial of Service — floods network resources | neptune, smurf, teardrop |
| `probe` | Scanning for vulnerabilities | portsweep, nmap, ipsweep |
| `r2l` | Remote to Local — unauthorized remote access | guess_passwd, ftp_write |
| `u2r` | User to Root — privilege escalation | buffer_overflow, rootkit |
